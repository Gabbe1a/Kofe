# Админка «Кофе Мама»: операционная модель

## Почему текущей панели недостаточно

В ТЗ меню, корзина и заказ зависят от выбранной точки. Текущий backend хранит
точку заказа, размеры, модификаторы, стоп-лист и настройки готовности, но
админ-экран позволяет менять только базовую цену. Это не покрывает сеть: у
точки может быть другая цена, отсутствие сиропа или временное меню.

## Рабочая структура

1. **Обзор** — сегодня/7 дней: заказы, выручка, средний чек, активные заказы;
   фильтр «вся сеть / конкретная точка».
2. **Заказы точки** — очередь `confirmed → preparing → ready → issued`, состав
   и модификаторы позиции; бариста видит только назначенные точки.
3. **Меню** — товары, категории, фото, базовая цена, размеры, КБЖУ,
   обязательные группы модификаторов. Создание товара — в одном мастере, а не
   по разрозненным таблицам.
4. **Точки** — адрес, телефон, координаты, часы, время приготовления, статус
   открытия. Внутри точки: локальная цена товара, доступность товара, сиропов,
   молока и других опций.
5. **Акции и медиа** — баннеры, порядок и период активности.
6. **Команда** — роли и доступ к точкам.

## Правила данных

- Базовая цена и набор модификаторов принадлежат товару.
- `venue_product_overrides` меняет только цену/доступность товара в конкретной
  точке; пустая цена означает «использовать базовую».
- `venue_modifier_option_overrides` отключает или переоценивает конкретный
  сироп/молоко только для выбранной точки.
- Стоп-лист остаётся быстрым временным выключателем; он не заменяет постоянную
  настройку ассортимента.
- В заказ всегда сохраняются снимки названий и цен, поэтому история не меняется
  после будущего редактирования меню.

## Права

| Раздел | admin | manager | barista |
|---|---|---|---|
| Сеть, точки, меню, медиа | да | да | нет |
| Локальные цены и наличие | да | только назначенные точки | только наличие |
| Очередь заказов | вся сеть | назначенные точки | назначенные точки |
| Аналитика | вся сеть | назначенные точки | нет |
| Сотрудники | да | нет | нет |
